3. Proyecto de Software

3.1 Fases del proyecto de software.

Para formular un proyecto de software es necesario definir las siguientes fases: Inicio, planeación, ejecución y cierre.

3.2 Formulación y gerencia del proyecto de software

Como en toda formulación de proyecto, para el desarrollo de un software se requiere realizar un ejercicio de pre estudio, donde se realiza un análisis y estudio requerido para poder definir e identificar la problemática a abordar, es por ello que entra en juego los aspectos de: los objetivos, la justificación, el alcance, los beneficios entre otros para poder determinar la mejor alternativa de solución.

3.3 Ciclo de vida del proyecto de software

El software como sistema, requiere ser desarrollado a través de una serie de actividades o etapas que conforman lo que se conoce como ciclo de vida, las cuales pueden variar de un autor a otro.

3.4. Metodología de marco lógico (MML)

El marco lógico, es una herramienta para la gerencia de proyectos usada en el diseño, planificación, ejecución y evaluación de proyectos de software.
